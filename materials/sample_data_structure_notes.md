# 数据结构考研演示知识库

说明：这是用于线上部署演示的小型知识库，覆盖栈、队列、顺序表、链表、快速排序、单链表逆置和二叉树遍历等常见考研知识点。正式使用时仍可在页面左侧上传更多 `.txt`、`.md`、`.pdf` 资料并重建知识库。

## 栈

栈是一种只允许在一端进行插入和删除操作的线性表。允许操作的一端称为栈顶，另一端称为栈底。栈的核心特点是后进先出，即 Last In First Out。常见基本操作包括初始化、判空、入栈、出栈、读取栈顶元素和销毁栈。栈常用于括号匹配、表达式求值、递归调用、深度优先搜索等场景。

## 队列

队列是一种只允许在一端插入、另一端删除的线性表。插入元素的一端称为队尾，删除元素的一端称为队头。队列的特点是先进先出，即 First In First Out。循环队列通常使用数组实现，通过取模运算复用存储空间。为了区分队空和队满，可以牺牲一个存储单元，令队满条件为 `(rear + 1) % maxSize == front`。

## 顺序表与链表

顺序表使用一组地址连续的存储单元保存线性表元素，支持随机访问，按下标访问元素的时间复杂度为 O(1)，但插入和删除通常需要移动元素，平均时间复杂度为 O(n)。链表通过指针或引用连接结点，不要求存储空间连续，插入和删除在已知结点位置时较方便，但按序号查找需要从头遍历，时间复杂度为 O(n)。顺序表适合查找频繁、长度变化较少的场景；链表适合插入删除频繁、长度变化较大的场景。

## 快速排序

快速排序是一种基于分治思想的排序算法。每一趟选择一个枢轴元素，将待排序序列划分为左右两个子序列，使左侧元素不大于枢轴，右侧元素不小于枢轴，然后递归排序左右子序列。快速排序平均时间复杂度为 O(n log n)，最坏时间复杂度为 O(n^2)，空间复杂度主要来自递归栈，平均为 O(log n)，最坏为 O(n)。当序列基本有序且枢轴选择不当时，快速排序容易退化。

## 单链表逆置

单链表逆置常用头插法或三指针迭代法。三指针法维护 `prev`、`cur`、`next` 三个指针，遍历链表时先保存当前结点后继，再将当前结点指向前驱，最后整体向后移动。该算法只扫描链表一次，时间复杂度为 O(n)，只使用常数个辅助指针，空间复杂度为 O(1)。

## 二叉树遍历

二叉树常见遍历方式包括先序遍历、中序遍历、后序遍历和层次遍历。先序遍历访问顺序为根、左子树、右子树；中序遍历访问顺序为左子树、根、右子树；后序遍历访问顺序为左子树、右子树、根。对于二叉搜索树，中序遍历可以得到递增有序序列。层次遍历通常借助队列实现。
